# Curatorr v0.1.7

## Summary

- Made Plex the recommended live playback source and added a settings toggle so admins can choose Plex or Tautulli webhooks for live ingestion.
- Rebuilt playback tracking around open-session accumulation so Curatorr can carry progress across pauses, resumes, scrobbles, track transitions, and missing duration metadata before writing a final play row.
- Reduced duplicate or stale recent-play history by tightening Tautulli gap-fill behavior and making Tautulli webhook handling follow the same finalization rules as Plex.
- Added automatic Plex webhook setup from the wizard and Settings, plus a popup completion page for the Plex auth flow.
- Improved Lidarr recommendation visibility and control with clearer dashboard wording, fixed queue reconciliation, separate automatic weekly caps, manual/automatic source labels, and Curatorr-managed Lidarr tags.

## Upgrade Notes

- Rebuild and redeploy the container so the new playback-session logic, webhook source setting, and Lidarr queue fixes are active.
- In `Settings -> General`, keep `Plex webhooks` selected as the live playback source unless you specifically want to compare Tautulli live behavior.
- Tautulli is still recommended as a backup and gap-fill source. Leave the daily sync enabled if you want missed Plex plays repaired without overwriting existing Plex-recorded rows.
- If Plex has not been re-authorized recently, use the Plex settings button to re-register the Curatorr webhook and confirm the server webhook list contains the secure Curatorr URL.
