# Curatorr v0.1.6

## Summary

- Hardened Plex and Tautulli webhook handling with shared-secret protected webhook URLs generated from Curatorr settings.
- Fixed Tautulli webhook auto-registration so the Settings button now writes the webhook URL, trigger selection, and JSON payload correctly.
- Restricted the Lidarr image proxy and several wizard/admin utility endpoints to reduce exposure of stored service credentials.
- Stopped rendering plaintext Plex, Tautulli, and Lidarr secrets to `co-admin` users, and moved credentialed Plex/Tautulli API calls away from query-string auth.

## Upgrade Notes

- Rebuild and redeploy the container so the new webhook validation and admin restrictions are active.
- Existing Plex and Tautulli webhook URLs must use the new secure Curatorr webhook URL with the `key=` parameter. The Tautulli Settings button can now register the Curatorr webhook correctly; Plex still needs a manual webhook URL update.
